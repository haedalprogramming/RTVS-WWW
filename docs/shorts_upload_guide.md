# 쇼츠 업로드 도구 사용 가이드 (`/upload.html`)

내부 전용 도구. 쇼츠 영상 1개를 올리면 YouTube(+추후 TikTok)에 한국어/영어 버전을 동시에 게시합니다.
전체 개발 로드맵과 상세 결정사항은 GitHub 이슈 [#35](https://github.com/haedalprogramming/RTVS-WWW/issues/35) 참고.

## 사용 방법

1. **접속**: https://roblox.code-builder.kr/upload.html
2. **비밀번호 입력** 후 로그인 — 세션은 12시간 유지되므로 같은 브라우저면 매번 다시 칠 필요 없음
3. **폼 작성**
   | 항목 | 필수 여부 | 설명 |
   |---|---|---|
   | 원본 영상 파일 | 필수 | mp4/mov/webm/mkv, 최대 2GB. 자막 없는 원본 그대로 업로드(가공 안 함) |
   | 게시할 플랫폼 | - | YouTube만 현재 사용 가능. TikTok은 앱 심사 대기 중이라 비활성화 표시됨 |
   | 게시 시점 | - | 지금 공개 / 지금 비공개(테스트용) / **예약 게시**(날짜·시간 지정 — 업로드는 바로 되고 지정 시각에 YouTube가 자동으로 공개 전환) |
   | 해시태그 | - | 기본값이 채워져 있음, 자유롭게 수정·삭제 가능 |
   | 한국어 제목·설명 | 필수 | |
   | 한국어 자막(.srt) | 선택 | YouTube 캡션 트랙으로 첨부(영상에 번인되지 않음, CC 버튼으로 켜야 보임) |
   | 영어 제목·설명 | 선택 | **비워두면 영어 버전 게시를 건너뜀** — 한국어만 급하게 올릴 때 유용 |
   | 영어 자막(.srt) | 선택 | |
4. **"게시" 클릭** → 진행률 표시 → 완료되면 플랫폼·언어별로 성공(영상 링크) / 실패 결과가 아래에 표시됨

### 원본 영상은 어디에 저장되나요?
업로드된 원본 영상은 임시로 Google Drive(기존 "영상 공유" 기능과 같은 Drive 폴더)에 자동 저장됩니다. 폴더 이름은 `shorts_YYYY-MM-DD_HH-MM-SS` 형식.

## 지금까지 개발된 것 (완료)

- 비밀번호 로그인 (12시간 세션 쿠키)
- 원본 영상 청크 업로드 → Google Drive 임시 저장
- **YouTube 업로드**: 한국어/영어 각각 별도 게시, 카테고리(게임)·아동용 아님·해시태그 고정 메타데이터 적용, `.srt` 캡션 트랙 첨부
- YouTube는 채널 소유자 OAuth 인증 완료 (브랜드 계정 "Roblox Code Builder"로 연결됨)
- 개인정보처리방침 / 이용약관 페이지 (TikTok 앱 등록에 필요해서 추가)
- 실제 영상으로 YouTube 한국어/영어 동시 게시 테스트 완료 (정상 동작 확인됨)
- **예약 게시** (YouTube): 업로드 시 `publishAt` 지정 → YouTube가 지정 시각에 자동 공개 전환

## 앞으로 개발할 것

| 항목 | 상태 |
|---|---|
| TikTok 개발자 앱 심사 | 제출 완료, **승인 대기 중** (리드타임 있음) |
| TikTok 업로드 함수 | 심사 통과 후 구현 예정 — 자막 없이 원본 영상 그대로 게시 |
| 업로드 실패 시 Slack 알림 + 오케스트레이션 정리 | 예정 |
| TikTok 예약 게시 | 예정 — TikTok API엔 예약 기능이 없어서, TikTok 업로드 함수(M5) 구현 후 별도 스케줄러(Vercel Cron)로 추가 예정 |

## 알려진 제한사항 / 주의사항

- 지금은 **YouTube만** 실제로 게시됩니다. TikTok 체크박스는 심사 통과 전까지 비활성화 상태
- 업로드 실패 시 **자동 재시도 없음** — 실패 메시지를 보고 직접 다시 시도해야 함 (중복 게시 방지 목적)
- YouTube 자막은 화면에 항상 보이는 게 아니라 시청자가 CC 버튼을 눌러야 나타남 (번인 아님)
- 세션이 12시간 유지되므로, 팀 공용 기기에서 사용 후 자리를 비울 때 주의
- YouTube Studio의 "관련 동영상" 카드(쇼츠 하단에 롱폼 영상을 연결하는 기능)는 Data API v3로 설정할 방법이 없음 — Studio 전용 수동 기능으로 확인됨(2026-07-14 확인). 필요하면 업로드 후 Studio에서 직접 연필 아이콘으로 설정
