# 로블록스 코드빌더 — 차시별 해설지 (교사용)

> 현재 코드에 실제로 구현되어 학생이 플레이 가능한 **로블록스1(기본) 12차시** 전체의 정답 예시와 해설을 정리한 문서입니다.

---

## 0. 이 문서 사용법

- 각 차시마다 **정답 예시**를 블록 순서대로 적어 두었습니다. 대부분의 맵은 **여러 가지 정답이 존재**합니다(특히 꽃밭류 맵). 여기 적힌 것은 "가장 표준적인 해설용 정답"이며, 학생이 다른 방식으로 통과해도 정답입니다.
- 블록 나열은 들여쓰기로 자식 블록(반복/조건/함수 안에 들어가는 블록)을 표현합니다.
  ```
  반복(5)
    장애물이 있으면
      오른쪽 회전
    앞으로 이동
  ```
  위 예시는 "반복" 블록 안에 "장애물이 있으면"과 "앞으로 이동"이 순서대로 들어있다는 뜻입니다.
- **좌표계**: 에이전트는 스폰 시 항상 -Z 방향을 정면으로 바라봅니다. `오른쪽 회전`을 1번 실행할 때마다 시계 방향으로 90도씩 회전합니다 (-Z → +X → +Z → -X → -Z 순서로 반복).

---

## 1. 블록 명령어 총정리 (7종)

| 표시명 | 커맨드(Type) | 카테고리 | 동작 설명 |
|---|---|---|---|
| 앞으로 이동 | `MoveForward` | Action | 현재 정면 방향으로 **8스터드** 전진. 벽에 막혀 2초 안에 도착하지 못하면 제자리로 복귀(막혔다는 뜻). 도착 시 그리드에 정확히 스냅됨 |
| 오른쪽 회전 | `TurnRight` | Action | 제자리에서 오른쪽(시계 방향)으로 **90도** 즉시 회전. 회전 후 0.3초 대기 |
| 꽃 심기 | `PlaceFlower` | Action | 현재 위치 바닥에 꽃(줄기+머리) 모델을 생성. 이동/회전과 달리 **제자리에서** 실행되며 몇 번을 실행해도 그 자리에 꽃이 계속 쌓임 |
| 반복 | `RepeatN` (파라미터 `Count`, 기본값 3) | Loop | 안에 들어있는 하위 블록들을 **Count번** 반복 실행. 1~20회 범위로 제한됨(20 초과 입력 시 자동으로 20으로 조정) |
| 장애물이 있으면 | `IfObstacle` | Conditional | 정면 9스터드 이내에 장애물(벽 등)이 있는지 레이캐스트로 검사. **참일 때만** 안에 들어있는 하위 블록을 실행하고, 거짓이면 건너뜀 |
| 함수 만들기 | `DefineFunc` (파라미터 `Name`) | FuncDef | 안에 들어있는 블록 묶음을 **실행하지 않고** 지정한 이름으로 저장만 함. 캔버스 어디에 있어도 실행 전에 먼저 등록됨(호이스팅) |
| 함수 호출 | `CallFunc` (파라미터 `Name`) | FuncCall | `함수 만들기`로 저장된 블록 묶음을 이름으로 찾아 그 자리에서 실행 |

**진행 조건(자동 감지) 2가지**
- **골 타일 터치**: 금색 타일에 에이전트가 닿으면 서버가 자동 감지하여 차시 완료 처리
- **꽃 개수 도달**: `꽃 심기` 블록이 실행될 때마다 서버가 해당 맵의 꽃 개수를 세어, 목표 수(goalTarget)에 도달하면 차시 완료 처리

## 2. 차시별 해설

### 1차시 — 코드빌더 시작하기 (맵 M-A · L자형)
**학습목표:** 블록 코드로 캐릭터를 이동시킬 수 있다 | **핵심 개념:** 순차구조 | **진행 조건:** 골 타일 터치

**맵 구조:** 스폰에서 정면(-Z)으로 2칸, 오른쪽으로 꺾어서 2칸 더 가면 골. 통로 폭 1칸의 벽으로 막혀 있어 정해진 경로로만 이동 가능.

**정답 예시**
```
앞으로 이동
앞으로 이동
오른쪽 회전
앞으로 이동
앞으로 이동
```

**해설:** 가장 기본적인 순차 실행 문제. "이동 2번 + 회전 1번 + 이동 2번"이 곧 명령 순서라는 것을 체감시키는 것이 목표입니다. 학생이 회전 방향(오른쪽=시계 방향)을 헷갈려 하면, 직접 제자리에서 몸으로 돌아보게 하면 이해가 빠릅니다.

---

### 2차시 — 미로 탈출하기 (맵 M-B · S자형)
**학습목표:** 순서에 맞게 블록을 나열하여 미로를 탈출할 수 있다 | **핵심 개념:** 순차 심화 | **진행 조건:** 골 타일 터치

**맵 구조:** 세로로 3칸(-Z) → 오른쪽 회전 → 가로로 3칸(+X) → 오른쪽 회전 → 세로로 3칸(다시 +Z 방향, 원래 진행 방향과 반대)까지 가면 골.

**정답 예시**
```
앞으로 이동
앞으로 이동
앞으로 이동
오른쪽 회전
앞으로 이동
앞으로 이동
앞으로 이동
오른쪽 회전
앞으로 이동
앞으로 이동
앞으로 이동
```

**해설:** 1차시보다 구간과 회전 횟수가 늘어난 심화 문제. 회전을 2번 하고 나면 처음 출발 방향과 반대(+Z)로 걷게 된다는 점을 짚어주면 좋습니다(-Z→+X→+Z).

---

### 3차시 — 꽃 한 송이 심기 (맵 F-A · 직선 3칸)
**학습목표:** 이동과 함께 꽃 심기 블록을 사용할 수 있다 | **핵심 개념:** `PlaceFlower` 도입 | **진행 조건:** 꽃 1송이 심기

**맵 구조:** 스폰 포함 일직선 3칸, 벽이 없는 열린 구간(꽃밭 계열 맵은 벽이 없습니다).

**정답 예시**
```
앞으로 이동
꽃 심기
```

**해설:** 목표는 꽃 1송이만 심으면 완료라서 사실 `꽃 심기` 한 블록만 실행해도 통과됩니다. 다만 "이동하면서 꽃을 심는다"는 이번 차시 목표를 살리기 위해 위 예시처럼 이동 1번 후 심는 것을 표준 해설로 안내하세요. (학생이 이동 없이 제자리에서 심어도 정답 처리됩니다 — 오답이 아님을 미리 알려주면 좋습니다.)

---

### 4차시 — 꽃길 만들기 (맵 F-B · 7×7 그리드)
**학습목표:** 이동하면서 꽃을 심어 패턴을 만들 수 있다 | **핵심 개념:** 순차 + 꽃 심기 조합 | **진행 조건:** 꽃 3송이 심기

**맵 구조:** 7×7 오픈 그리드(테두리는 울타리), 스폰은 남서쪽 안쪽 모서리.

**정답 예시**
```
앞으로 이동
꽃 심기
앞으로 이동
꽃 심기
앞으로 이동
꽃 심기
```

**해설:** "이동 → 심기"를 세 번 반복해 일직선 꽃길을 만드는 패턴 문제입니다. 그리드가 열려 있어 어느 방향으로 심어도 3송이만 채우면 통과되므로, 자유롭게 배치하는 학생도 정답입니다. 다음 차시(반복문)로 넘어가기 전에 "같은 동작을 세 번 반복해서 썼다"는 점을 짚어 반복문의 필요성을 예고하면 좋습니다.

---

### 5차시 — 반복으로 미로 탈출 (맵 M-C · 직선 6칸)
**학습목표:** 반복 블록으로 같은 동작을 여러 번 실행할 수 있다 | **핵심 개념:** 반복문 | **필수 블록:** `RepeatN` | **진행 조건:** 골 타일 터치

**맵 구조:** 스폰~골까지 일직선 통로, 총 5칸 전진해야 도달.

**정답 예시**
```
반복(5)
  앞으로 이동
```

**해설:** `앞으로 이동`을 5번 나열하는 대신 `반복(5)` 안에 1개만 넣으면 동일하게 동작한다는 것을 보여주는 차시. `Count` 파라미터를 5로 바꾸는 것을 꼭 짚어주세요(기본값은 3이라 그대로 두면 통과하지 못합니다).

---

### 6차시 — 반복으로 꽃밭 만들기 (맵 F-C · 직선 5칸 꽃밭)
**학습목표:** 반복 블록으로 규칙적인 꽃밭을 만들 수 있다 | **핵심 개념:** 반복 + 꽃 심기 조합 | **필수 블록:** `RepeatN` | **진행 조건:** 꽃 5송이 심기

**정답 예시**
```
반복(5)
  꽃 심기
  앞으로 이동
```

**해설:** "심고 → 이동"을 반복하면 일정한 간격으로 꽃 5송이가 심어집니다. 순서를 반대로(`앞으로 이동` → `꽃 심기`) 써도 결과적으로 꽃 5송이는 채워지므로 정답 처리되지만, 첫 칸(스폰 위치)에는 꽃이 심어지지 않는 차이가 생깁니다 — 두 방식의 결과 차이를 비교해 보여주면 좋은 심화 발문이 됩니다.

---

### 7차시 — 장애물 감지하기 (맵 M-D · T자형)
**학습목표:** 조건 블록으로 장애물을 감지하고 행동을 바꿀 수 있다 | **핵심 개념:** 조건문 | **필수 블록:** `IfObstacle` | **진행 조건:** 골 타일 터치

**맵 구조:** 스폰에서 1칸 전진하면 막다른 T자 분기점. 정면은 벽으로 막혀 있고 오른쪽으로 꺾으면 바로 골.

**정답 예시**
```
앞으로 이동
장애물이 있으면
  오른쪽 회전
앞으로 이동
```

**해설:** 분기점에서 정면에 벽이 있는지 `장애물이 있으면`으로 검사해 참일 때만 회전하고, 그 다음 다시 전진해서 골로 들어갑니다. `장애물이 있으면` 블록 안에 `앞으로 이동`까지 넣어버리는 실수가 흔한데, 그렇게 하면 회전과 동시에 이동해버려 원하는 결과가 나오지 않는다는 점(회전 다음 이동은 조건 블록 밖에 별도로 있어야 함)을 짚어주세요.

---

### 8차시 — 반복 속 조건 (맵 M-E · L자형, 7차시와 다른 자리에 재사용)
**학습목표:** 반복 안에 조건을 넣어 복잡한 동작을 만들 수 있다 | **핵심 개념:** 반복 + 조건 중첩 | **필수 블록:** `RepeatN`, `IfObstacle` | **진행 조건:** 골 타일 터치

**맵 구조:** 1차시(M-A)와 동일한 L자형(앞×2 → 오른쪽 → 앞×2)이지만, 이번엔 "벽을 만나면 자동으로 꺾는" 범용 알고리즘으로 풀도록 유도합니다.

**정답 예시**
```
반복(4)
  장애물이 있으면
    오른쪽 회전
  앞으로 이동
```

**해설:** 매 반복마다 "정면에 벽이 있으면 먼저 오른쪽으로 돈 다음, 무조건 한 칸 전진"하는 동작을 4번 반복합니다. 1~2번째 반복에서는 장애물이 없어 그냥 전진하고, 3번째 반복에서 막다른 벽을 만나 회전 후 전진, 4번째 반복에서 마지막 한 칸을 이동해 골에 도착합니다. 1차시의 "앞×2, 오른쪽, 앞×2"를 그대로 써도 통과되지만, 이번 차시의 목적은 **어떤 모양의 통로든 통하는 범용 규칙**을 만드는 것이므로 위 정답을 표준으로 안내하세요.

---

### 9차시 — 나만의 함수 만들기 (맵 F-D · 7×7 그리드)
**학습목표:** 함수를 정의하고 호출하여 코드를 재사용할 수 있다 | **핵심 개념:** 함수 정의·호출 | **필수 블록:** `DefineFunc`, `CallFunc` | **진행 조건:** 꽃 3송이 심기

**정답 예시**
```
함수 만들기("꽃심고전진")
  앞으로 이동
  꽃 심기
함수 호출("꽃심고전진")
함수 호출("꽃심고전진")
함수 호출("꽃심고전진")
```

**해설:** `함수 만들기` 블록 자체는 실행되지 않고 이름만 등록한다는 점이 핵심입니다(캔버스에 올려두기만 해도 됨). 그 아래에서 `함수 호출`을 3번 쓰면 "이동+심기" 동작이 3번 반복되어 꽃 3송이가 채워집니다. 6차시의 `반복(3) { 앞으로 이동, 꽃 심기 }`와 결과는 같지만, "이름 붙여 재사용"하는 함수의 개념 차이를 비교해서 설명해주면 좋습니다.

---

### 10차시 — 함수로 꽃길 만들기 (맵 F-E · 직선 5칸 꽃밭)
**학습목표:** 함수와 반복 블록을 함께 사용하여 자동으로 꽃길을 만들 수 있다 | **핵심 개념:** 함수 + 반복 응용 | **필수 블록:** `CallFunc` | **진행 조건:** 꽃 5송이 심기

**정답 예시**
```
함수 만들기("꽃심고전진")
  앞으로 이동
  꽃 심기
반복(5)
  함수 호출("꽃심고전진")
```

**해설:** 9차시(함수)와 5·6차시(반복)를 합친 문제입니다. 함수 안의 동작을 반복문으로 여러 번 호출할 수 있다는 것, 즉 함수 호출도 일반 블록처럼 반복문 안에 넣을 수 있다는 점을 강조하세요.

---

### 11차시 — 함수 여러 개 조합하기 (맵 F-D2 · 7×7 그리드)
**학습목표:** 여러 함수를 만들고 조합하여 복잡한 동작을 만들 수 있다 | **핵심 개념:** 함수 심화 | **필수 블록:** `CallFunc` | **진행 조건:** 꽃 6송이 심기

**정답 예시**
```
함수 만들기("직진꽃")
  앞으로 이동
  꽃 심기
함수 만들기("꺾어서꽃")
  오른쪽 회전
  앞으로 이동
  꽃 심기
함수 호출("직진꽃")
함수 호출("직진꽃")
함수 호출("꺾어서꽃")
함수 호출("직진꽃")
함수 호출("직진꽃")
함수 호출("꺾어서꽃")
```

**해설:** 서로 다른 동작을 하는 함수 2개(`직진꽃`, `꺾어서꽃`)를 정의하고, 이를 원하는 순서로 조합해서 부른 것이 정답 예시입니다. 함수를 1개만 써서 6번 호출해도(9·10차시 방식) 목표(꽃 6송이)는 달성되므로 오답은 아니지만, 이번 차시의 학습목표(**여러 함수 조합**)를 보여주려면 위처럼 2개 이상의 함수를 섞어 쓰는 것을 표준 해설로 안내하세요.

---

### 12차시 — 나만의 정원 만들기 (맵 OW · 9×9 오픈월드)
**학습목표:** 배운 모든 블록을 활용하여 나만의 정원을 창작할 수 있다 | **핵심 개념:** 자유 프로젝트 | **진행 조건:** 꽃 10송이 심기

**정답 예시 (7개 블록 총정리용)**
```
함수 만들기("전진꽃")
  앞으로 이동
  꽃 심기
반복(10)
  장애물이 있으면
    오른쪽 회전
  함수 호출("전진꽃")
```

**해설:** 특정 정답이 없는 자유 창작 차시입니다. 위 예시는 "울타리에 막히면 자동으로 방향을 바꾸며 꽃 10송이를 채우는" 범용 알고리즘으로, 그동안 배운 7개 블록(이동·회전·꽃심기·반복·조건·함수정의·함수호출)을 모두 한 번씩 사용해 총정리하는 형태입니다. 학생에게는 이 예시를 강요하지 말고, 꽃 10송이만 채우면 어떤 방식이든 정답이라는 점을 강조해주세요.

## 3. 참고 사항

- **1차시 체험수업(50분, 6~9세 대상)**: 별도의 맵 코드는 없고, 위 12차시 중 **1차시(M-A, 미션)** 와 **4차시(F-B, 꽃길)** 맵을 그대로 활용해 미션 체험 + 자유 창작 흐름으로 진행합니다. 해설은 위 1차시·4차시 항목을 참고하세요.
- **로블록스2(심화 12차시)**: `왼쪽 회전`, `변수에 저장`, `변수 읽기`, `장애물 없는 동안 반복` 등 신규 블록 4개가 계획되어 있으나 아직 코드로 구현되지 않아 이번 해설지에는 포함하지 않았습니다. 구현 완료 후 별도 해설지를 추가할 예정입니다.
- 맵/블록 사양은 `src/shared/LessonConfig.luau`, `src/server/MapBuilder.luau`, `src/blockdefs/*.luau` 코드 기준으로 작성했습니다. 추후 맵이나 목표 수치가 변경되면 이 문서도 함께 업데이트해야 합니다.
